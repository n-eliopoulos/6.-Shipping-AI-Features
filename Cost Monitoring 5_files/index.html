Cost Optimization Levers

You don't need to sacrifice quality to cut costs. Four levers can reduce AI spend by 30–60% without degrading output quality.

The four levers

LeverSavingsEffortRisk
Model selection — Use Haiku for simple tasks, Sonnet for complex50–90%LowQuality may drop for edge cases
Prompt trimming — Remove verbose system prompts, redundant context10–30%LowMinimal if done carefully
Response caching — Cache identical or near-identical queries20–50%MediumStale results for dynamic data
Batching — Combine multiple small requests into one10–20%MediumLatency increase for individual items

Example: $500/day → $180/day

# Before: All calls use Sonnet, verbose prompts, no caching
# Daily cost: $500

# Optimization 1: Route simple classifications to Haiku
# 60% of calls are simple → saves $180/day

# Optimization 2: Cache repeated entity lookups
# 25% of calls are duplicates → saves $75/day

# Optimization 3: Trim system prompt from 800 to 200 tokens
# Saves ~15% on input costs → saves $45/day

# After: $500 - $180 - $75 - $45 = $200/day (60% reduction)
Try it now: In the sandbox, take a feature costing $500/day. Apply three optimizations: switch low-complexity calls to Haiku, cache repeated entity lookups, and trim verbose system prompts. Measure the new cost. Target: under $200/day.
Model selection is the biggest lever. If 60% of your calls don't need Sonnet-level reasoning, switching them to Haiku cuts your bill in half overnight.
