Graceful Degradation in Practice

Putting it all together: a compliance screening feature that handles every failure mode gracefully. Fallback chain + rate limiter + circuit breaker + timeout handler, wired together into a single resilient system.

The full resilience stack

Request
  → Rate limiter (queue if over limit)
    → Budget check (reject if exhausted)
      → Circuit breaker (fallback if open)
        → Timeout wrapper (graceful timeout at 10s)
          → Primary model (Claude Sonnet)
            ↓ fails → Fallback model (Claude Haiku)
              ↓ fails → Cached response
                ↓ no cache → Static fallback

Chaos scenario walkthrough

Resilience isn't one pattern — it's the composition of many patterns working together. Each layer handles a specific failure mode. Together, they ensure the user always gets a response.
