Token Usage Tracking

Every AI call returns token counts. Capture them, tag them by feature and caller, and store them for trend analysis. The token counter is middleware — it wraps every AI call automatically.

Token tracking middleware

class TokenTracker:
    def __init__(self):
        self.usage = []  # list of usage records

    def track(self, model, feature, user_id, input_tokens, output_tokens):
        self.usage.append({
            "timestamp": time.time(),
            "model": model,
            "feature": feature,
            "user_id": user_id,
            "input_tokens": input_tokens,
            "output_tokens": output_tokens,
        })

    def totals_by_feature(self):
        by_feature = {}
        for u in self.usage:
            f = u["feature"]
            if f not in by_feature:
                by_feature[f] = {"input": 0, "output": 0}
            by_feature[f]["input"] += u["input_tokens"]
            by_feature[f]["output"] += u["output_tokens"]
        return by_feature

What to aggregate

  • By feature: "Trade surveillance used 12M tokens this week"
  • By user: "Analyst team A used 3× more than team B"
  • By model: "80% of tokens went to Sonnet, 20% to Haiku"
  • Over time: "Token usage grew 40% month-over-month"
Try it now: In the sandbox, build a token tracking middleware that wraps ai_shim. Every call records input/output tokens, model, and caller identity. Aggregate totals by feature after 10 calls.
Token tracking is the foundation of cost management. If you can't count tokens per feature, you can't attribute costs, set budgets, or detect anomalies.
