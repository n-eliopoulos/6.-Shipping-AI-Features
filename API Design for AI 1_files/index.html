Module 1: API Design for AI

AI Endpoints Are Different

Traditional REST endpoints are fast, deterministic, and cacheable. An AI endpoint is none of these things. Understanding the differences is the first step to designing APIs that don't frustrate users or bankrupt your team.

PropertyTraditional RESTAI Endpoint
Latency50–200ms2–30 seconds
DeterminismSame input → same outputSame input → different output
Cost per callFractions of a cent$0.01–$1.00+
CacheabilityHigh (ETags, TTLs)Low (non-deterministic)
Failure modesTimeouts, 5xxTimeouts, 5xx, rate limits, hallucinations, budget exhaustion

What this means for API design at FINRA

  • Timeouts need rethinking. A 30-second default timeout will kill half your AI calls. You need separate timeout strategies.
  • Rate limiting is non-negotiable. One runaway client can exhaust your Bedrock quota and block every team.
  • Streaming changes the contract. Users can't wait 10 seconds staring at a spinner. Token-by-token delivery is a UX requirement.
  • Every call costs money. Input validation before the AI call isn't just good practice — it's cost protection.
Design AI endpoints as a separate class of service. The patterns you use for CRUD APIs will fail for AI — different timeouts, different error handling, different cost model.
