Module 3: Cost Monitoring

Cost Attribution

Knowing total AI spend is not enough. You need to know: "Trade surveillance costs $X/month, compliance screening costs $Y/month." Cost attribution maps token usage to dollar amounts, broken down by feature, team, and model.

From tokens to dollars

PRICING = {
    "claude-sonnet": {"input": 3.00 / 1_000_000, "output": 15.00 / 1_000_000},
    "claude-haiku":  {"input": 1.00 / 1_000_000, "output":  5.00 / 1_000_000},
}

def compute_cost(model, input_tokens, output_tokens):
    rates = PRICING[model]
    return (input_tokens * rates["input"]) + (output_tokens * rates["output"])

Cost allocation tags

Every AI call should carry tags that enable attribution:

  • feature — Which product feature triggered this call
  • team — Which team owns the feature
  • model — Which model was used
  • environment — Production vs. staging vs. development

Chargeback models

For shared AI infrastructure, teams need to see their share of the bill. Two common models:

  • Direct attribution: Each team pays for their tagged usage. Simple, fair, but requires consistent tagging.
  • Proportional allocation: Shared costs (platform overhead) split by usage percentage. More complex but accounts for shared infrastructure.
Interactive: The cost attribution dashboard below shows a pie chart and table with cost breakdown by feature, team, and model. Run calls tagged with different features and watch the attribution update in real time.
Cost attribution turns "AI costs $20K/month" into "trade surveillance costs $8K, compliance screening costs $6K, and report generation costs $6K." That's the difference between a budget line item and an actionable optimization target.
