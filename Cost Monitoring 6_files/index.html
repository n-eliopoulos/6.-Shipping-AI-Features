Module 3: Cost Monitoring

Exercise: Build a Cost-Tracking Dashboard

Instrument a multi-feature AI system with full cost tracking, attribution, and budget enforcement. The system has 3 features using 2 models.

The system

  • Trade surveillance — Uses Sonnet for complex risk analysis
  • Compliance screening — Uses Haiku for quick classification, Sonnet for detailed review
  • Report generation — Uses Sonnet for narrative summaries

Requirements

  1. Token tracking — Record input/output tokens for every call, tagged by feature and model.
  2. Cost attribution dashboard — Show per-feature and per-model cost breakdown. Pie chart + table.
  3. Budget alerts — Configure daily budgets per feature. Warning at 80%, critical at 95%, hard stop at 100%.
  4. Anomaly detection — Simulate a prompt injection causing 10× token usage. Verify the alert and hard stop work.

Validation criteria

  • Cost attribution matches expected values within 5%
  • Budget alert fires at the correct threshold
  • Hard stop prevents overspend
  • Anomaly (10× token spike) is detected and stopped
In the sandbox: Build the tracking, dashboard, and alerting. Run the simulation and verify all criteria pass.
A cost dashboard that nobody checks is useless. Budget enforcement with hard stops is the only reliable way to prevent runaway AI costs.
