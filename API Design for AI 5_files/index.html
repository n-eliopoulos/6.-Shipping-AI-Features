Request Validation and Rate Limiting

Every AI call costs money. Validate inputs before sending them to the model. Reject oversized requests, enforce rate limits, and estimate token costs upfront. At FINRA scale, one unvalidated endpoint can burn through your monthly Bedrock budget in hours.

Pre-call validation checklist

  • Token estimation. Estimate input tokens (~4/3 tokens per word). Reject requests exceeding your budget per call.
  • Input sanitization. Strip unnecessary whitespace, validate required fields, check for prompt injection patterns.
  • Content length limits. Set max input size per endpoint — a 100K-token request costs 100× more than a 1K-token request.

Rate limiting strategies

# Token bucket rate limiter
class RateLimiter:
    def __init__(self, max_requests, window_seconds):
        self.max_requests = max_requests
        self.window = window_seconds
        self.requests = {}  # user_id → [timestamps]

    def allow(self, user_id):
        now = time.time()
        user_reqs = [t for t in self.requests.get(user_id, [])
                     if now - t < self.window]
        if len(user_reqs) >= self.max_requests:
            return False
        user_reqs.append(now)
        self.requests[user_id] = user_reqs
        return True
