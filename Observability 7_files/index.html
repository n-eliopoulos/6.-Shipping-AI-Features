Exercise: Instrument an AI Feature

You're given a pre-built trade surveillance AI feature with 3 chained AI calls: extract entities → assess risk → generate report. Your job: add full observability.

Requirements

  1. Structured logging — Every AI call produces a JSON log entry with all required fields (timestamp, request ID, correlation ID, model, tokens, latency, status, cost).
  2. Metrics collection — Compute and display the four core metrics: token usage, latency percentiles, error rate, and cost.
  3. Trace aggregation — All 3 calls in a workflow share a correlation ID. The trace shows parent/child spans with timing.
  4. Alerting — Configure alerts for latency spikes and cost anomalies. Verify they fire on injected anomalies.

Validation

Run 20 simulated requests through the instrumented feature. The system checks:

  • All calls logged with required fields (no missing fields)
  • Metrics dashboard shows correct aggregated values
  • Traces link all 3 calls per workflow correctly
  • Alerts fire on the injected latency spike and cost anomaly
In the sandbox: The pre-built feature code is provided. Add your instrumentation layer, run the simulation, and inspect the results.
Observability isn't a feature you add later — it's infrastructure you build from the start. If you can't see what your AI is doing, you can't fix it when it breaks.
