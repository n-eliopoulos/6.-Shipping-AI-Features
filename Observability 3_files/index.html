Core Metrics

Four metrics you must track for every AI feature in production. Everything else is optional — these are not.

The four core AI metrics

MetricWhat to trackWhy it matters
Token usageInput/output tokens per call, aggregated by featureDrives cost, predicts budget burn rate
Latencyp50, p95, p99 per endpointUser experience, timeout tuning
Error rateBy error type (429, 500, timeout, malformed)Reliability, fallback trigger tuning
CostPer call, per user, per feature, daily/monthlyBudget management, chargeback

Computing metrics from structured logs

# From a stream of log entries, compute running metrics
def compute_metrics(log_entries):
    latencies = [e["latency_ms"] for e in log_entries]
    return {
        "total_input_tokens": sum(e["input_tokens"] for e in log_entries),
        "total_output_tokens": sum(e["output_tokens"] for e in log_entries),
        "p50_latency_ms": percentile(latencies, 50),
        "p95_latency_ms": percentile(latencies, 95),
        "p99_latency_ms": percentile(latencies, 99),
        "error_rate": sum(1 for e in log_entries if e["status"] != "success") / len(log_entries),
        "total_cost_usd": sum(e["cost_usd"] for e in log_entries),
    }
