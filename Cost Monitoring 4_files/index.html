Budget Alerting

Set daily, weekly, and monthly budgets per feature. Alert at 80% (warning), 95% (critical), and 100% (hard stop). A hard stop that actually stops spending is the only thing between you and a surprise bill.

Budget enforcement tiers

ThresholdActionUser impact
80% (warning)Alert to Slack/email, log warningNone — informational
95% (critical)Page on-call, escalate to team leadNone — but action needed soon
100% (hard stop)Reject new AI calls, return cached/fallbackDegraded service with graceful message

Implementation pattern

class BudgetEnforcer:
    def __init__(self, daily_budget_usd):
        self.daily_budget = daily_budget_usd
        self.spent_today = 0.0

    def check(self, estimated_cost):
        ratio = (self.spent_today + estimated_cost) / self.daily_budget
        if ratio >= 1.0:
            return {"allowed": False, "reason": "Daily budget exhausted",
                    "message": "Service temporarily limited. Try again tomorrow."}
        if ratio >= 0.95:
            self.alert("critical", f"Budget at {ratio:.0%}")
        elif ratio >= 0.80:
            self.alert("warning", f"Budget at {ratio:.0%}")
        self.spent_today += estimated_cost
        return {"allowed": True}

A budget alert without a hard stop is just a notification you'll ignore at 2 AM. Implement actual enforcement — reject calls when the budget is exhausted and return a graceful fallback.
