Structured Logging for AI

Every AI call should produce a JSON log entry with: timestamp, request ID, model, input tokens, output tokens, latency, status, and cost estimate. Not print statements — structured, queryable, machine-readable logs.

The minimum viable AI log entry

{
  "timestamp": "2025-04-29T14:32:01.234Z",
  "request_id": "req-7f3a2b",
  "correlation_id": "workflow-abc-123",
  "model": "claude-sonnet-4-20250514",
  "input_tokens": 1250,
  "output_tokens": 340,
  "latency_ms": 2847,
  "status": "success",
  "cost_usd": 0.00885,
  "feature": "trade-surveillance",
  "user_id": "analyst-42"
}

Correlation IDs for multi-step workflows

A single user request may trigger 3 AI calls: classify → retrieve → summarize. A correlation_id links them together so you can trace the full workflow from a single ID.

def log_ai_call(request_id, correlation_id, model, input_tokens,
                output_tokens, latency_ms, status, feature):
    entry = {
        "timestamp": datetime.utcnow().isoformat() + "Z",
        "request_id": request_id,
        "correlation_id": correlation_id,
        "model": model,
        "input_tokens": input_tokens,
        "output_tokens": output_tokens,
        "latency_ms": latency_ms,
        "status": status,
        "cost_usd": compute_cost(model, input_tokens, output_tokens),
        "feature": feature,
    }
    print(json.dumps(entry))
Try it now: In the sandbox, implement a structured logger that wraps ai_shim calls. Every call produces a JSON log entry with all required fields. Run 5 calls and inspect the log output.
Alternative considered: Use print() or unstructured log lines like logger.info(f"Called {model}, got {response}"). This is fast to write and impossible to query at scale. Every dashboard, alert rule, and trace correlation you'll want to build downstream needs structured fields — you cannot grep your way to a 99th-percentile latency chart over six months of free-form text. The structured-from-day-one cost is a few extra fields per log line; the cost of converting later is rewriting every log statement in your codebase.
Structured logging is the foundation of AI observability. Every metric, dashboard, alert, and trace is built from these log entries. Get the schema right from day one.
