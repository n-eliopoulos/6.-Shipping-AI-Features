Module 4: Error Handling and Degradation

Fallback Strategies

When the primary model fails, don't return an error — fall back. A 3-tier fallback chain ensures users always get a response, even if it's degraded.

The fallback ladder

Tier 1: Claude Sonnet (full analysis)
  ↓ fails
Tier 2: Claude Haiku (simplified prompt, faster, cheaper)
  ↓ fails
Tier 3: Cached response from last successful call
  ↓ no cache
Tier 4: Static fallback ("Service temporarily limited. Here's what we know...")

Feature-level degradation

Service levelWhat the user getsWhen
FullComplete risk analysis with citationsPrimary model available
DegradedSummary-only analysis, no citationsPrimary down, fallback model available
MinimalLast known analysis + "results may be stale" warningAll models down, cache available
Unavailable"Service temporarily limited" with ETAEverything down
