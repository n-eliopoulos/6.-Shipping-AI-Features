Exercise: Design an AI API

Design a complete API for a compliance document screening service at FINRA. This service receives regulatory filings, classifies them by type, performs detailed analysis, and generates summaries.

Requirements

Define three endpoints:

  1. Synchronous classification — Quick categorization of a document (risk level, document type). Should return in <5 seconds.
  2. Async detailed analysis with polling — Deep analysis that may take 30–60 seconds. Submit, get a job ID, poll for results.
  3. Streaming summary — Generate a human-readable summary with tokens streamed as they're produced.

For each endpoint, define:

  • Request schema (required fields, optional parameters)
  • Response schema (success and error cases)
  • Timeout configuration (client, server, model)
  • Rate limit rules (requests/minute, tokens/minute)
  • Input validation rules (max size, required fields)
In the sandbox: Build out the endpoint schemas, timeout values, and rate limit rules. The system validates completeness and checks for common mistakes — missing error responses, no timeout config, no rate limits.
A well-designed AI API anticipates failure at every layer. If your schema doesn't include error responses, timeout values, and rate limits, it's not production-ready.
